**Introduce a provider-agnostic `IntentLlm` interface for the LLM-backed translation/orchestration steps of the RFC 9315 cycle (business-intent agent), with `ClaudeClient` as the default Anthropic adapter and a deterministic, network-free `StubIntentLlm`, selected at the composition root via `LLM_PROVIDER`.**

## 3. Context and Problem Statement

The business-intent agent welded the BSS phase strategy to a concrete `ClaudeClient` (Anthropic SDK). Two problems followed: (a) the intent cycle — and therefore the **TMF921 CTK + canonical O2C** — could not run without Anthropic API credits, blocking the 005 cutover's **Gate B** in credit-constrained / CI / offline environments; (b) the provider could not be switched without editing the cycle.

**Problem statement**: How do we make the LLM provider switchable and let the conformance suite run without API credits, without changing the conformance-bearing behaviour of the default path?

**Requirements/links**: enables `ARC-005-ADR-002` Gate B (deferred-tag runbook §10); aligns PRIN 10 (Loose Coupling), PRIN 14 (Maintainability), PRIN 4 (no secret required to boot the stub path).

---

## 4. Decision Drivers

- **Switchability** (user requirement: "LLM-agnostic so can switch") — ports-and-adapters seam.
- **Offline conformance** — run the CTK/O2C without credits (Gate B proxy).
- **Behaviour preservation** — the default Anthropic path must be unchanged (no conformance drift).
- **Boot without a secret in stub mode** (PRIN 4 — no key needed for the stub).

| Principle | Alignment |
|-----------|-----------|
| PRIN 10 — Loose Coupling | ✅ Interface seam; cycle depends on `IntentLlm`, not a provider |
| PRIN 14 — Maintainability | ✅ New provider = one file + one factory branch |
| PRIN 3 — Standards Conformance | ⚠️ Stub is a *regression proxy*, not the real-translation conformance claim (see §7) |

## 5. Considered Options

### Option 1: `IntentLlm` interface + Anthropic adapter + Stub, env-selected — CHOSEN

Extract a 2-method interface (`analyzeIntent`, `generateOffer`); `ClaudeClient implements IntentLlm` (unchanged); add `StubIntentLlm` (deterministic, no SDK/key); select via `LLM_PROVIDER` at the composition root, building Anthropic lazily.

- ✅ Switchable; offline-capable; default path untouched; stub boots keyless.
- ✅ Verified: app typecheck clean; **353/353 tests pass**; stub smoke deterministic.
- ❌ Stub output is canned (not real translation) — must be labelled in any evidence.

### Option 2: Keep `ClaudeClient`, add an env "mock mode" inside it

- ❌ Conflates provider with mock; SDK still imported/keyed; not a clean seam; not switchable to other real providers.

### Option 3: Do Nothing

- ❌ Cutover Gate B stays blocked without credits; no provider switching. Rejected.

---

## 6. Decision Outcome

**Option 1.**

> **In the context of** an RFC 9315 business-intent cycle welded to Anthropic,
> **facing** a credit-blocked conformance gate and no provider switchability,
> **we decided for** a provider-agnostic `IntentLlm` seam with a deterministic stub selected by `LLM_PROVIDER`,
> **to achieve** switchable providers and offline CTK/O2C runs with the default path unchanged,
> **accepting** that the stub is a regression proxy, not the full real-translation conformance claim.

### Positive

- ✅ `LLM_PROVIDER=stub` runs the full cycle (and CTK/O2C) with no SDK/key/network.
